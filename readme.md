# OC 桌宠使用说明

## 1. 资源准备
- 必须放在程序同目录：
  - `normal1.png`（闭嘴/静止）
  - `normal2.png`（张嘴/说话）
  - `oc.ico`（程序图标）

### 情绪表情（可选）
- 可在 `settings.json -> images` 配置以下键位，让不同情绪播放不同表情动画：
  - `happy_closed` / `happy_open` / `happy_blink`
  - `sad_closed` / `sad_open` / `sad_blink`
- 动画触发点：待机、说话开合嘴、随机眨眼都会按当前情绪取图。
- 未提供情绪图时会自动回退到已有图，不会报错。

### 单图模式
- 只放一张角色图也可以运行。
- 如果找不到 `normal1.png`，程序会自动使用目录里的第一张图片作为角色图。
- 如果找不到 `normal2.png`，程序会自动复用同一张图作为说话状态。

## 2. 首次安装
双击 `install.bat`
- 安装脚本会优先读取 `requirements.txt` 安装依赖，并补充安装桌宠核心依赖。
- 安装日志会写入 `logs/install_时间戳.log`，若安装异常请优先查看该日志。

## 2.1 若提示缺少 VC 运行库
- 双击 `download_vc_redist.bat` 下载最新版 VC++ 2015-2022 运行库（x64 + x86）
- 下载完成后可按提示一键静默安装

## 2.5 自动生成三张表情图（推荐）
- 把你的原图放到程序目录，命名为 `source.png`（也支持 jpg/jpeg/webp）。
- 双击 `make_assets.bat`。
- 会自动生成：
  - `normal1.png`（闭嘴）
  - `normal2.png`（张嘴）
  - `blink1.png`（眨眼）

## 3. 运行
双击 `run.bat`
- 启动日志会写入 `logs/run_时间戳.log`。

## 3.1 调整桌宠大小
- 右键桌宠 -> `宠物大小`，拖动滑条后保存（60%~180%）。
- 也可手动改 `settings.json -> ui -> pet_scale`（例如 `1.2` 表示 120%）。
- 保存后会自动重载图片并生效。

## 3.2 安装/启动失败如何排查
- `install.bat` 日志：`logs/install_时间戳.log`
- `run.bat` 日志：`logs/run_时间戳.log`
- `download_vc_redist.bat` 若下载/安装失败，会在命令行直接显示失败阶段和返回码

## 4. 配置 API
- 启动后会生成/读取 `settings.json`
- 你可以直接编辑 `settings.json` 里的 `api_key`
- 或在 `oc.py` 搜索 `api输入口`

## 5. 右键菜单功能
- 历史记录（支持删除）
- 帮助/说明（读取本文件）
- 修改人设 Prompt（即时生效）
- 御主档案（注入对话前置设定）
- 传送门（网站快捷打开）
- 专注时钟与监督
- 背包/喂食、商城与金币循环
- 日程提醒、随机事件
- 观察日记（最近3天交互生成）
- 管家服务（启动本地程序）
- 猜拳/掷骰子小游戏
- 随手记、文件整理
- **iGEM助手**（组会记录 / 生信工作流 / 文档中心 / 任务看板 / 团队管理）

## 6. 对话交互规则
- 角色说话时自动开合嘴（normal1/normal2）
- 空闲状态随机眨眼（blink1）
- 文本为打字机显示
- 点击对话框可立即显示当前句完整内容
- 完整显示后，需等待 0.5 秒才能点下一句
- 每句完整显示后停留 2 秒
- 最后一句停留 3 秒后淡出

## 7. 系统与陪伴能力
- 整点报时（每小时触发）
- 昼夜作息（23:00-07:00 进入睡眠模式）
- 夜间可通过输入"叫醒/起床/wake up"等关键词唤醒，默认保持清醒 30 分钟
- CPU/内存监控（需要 `psutil`）
- 网络延迟监控（ping）
- 剪贴板互动：复制新文本时会出现"你复制了什么？"提示，点击可读取并吐槽；可"吃掉"清空剪贴板并增加好感
- 游戏进程检测与静音模式
- 媒体/音乐嗅探（Windows 媒体会话优先，浏览器标题兜底），检测到换歌后会生成"听歌感想"

## 8. 进阶已实装
- 长期记忆检索：程序会把每轮对话自动生成摘要并向量化存入 `memory.json`，当识别到"新话题"时，会先检索最相似的历史记忆并注入到 Prompt 作为上下文。
- 观察日记生成：右键菜单 `观察日记`，可按最近3天日志自动生成并保存到 `notes.txt`。

## 9. 本地语音朗读（TTS）
- 已接入 `pyttsx3` 轻量本地语音。
- 默认已开启（可在右键菜单随时关闭）。
- 右键菜单可用 `语音朗读开关` 开启/关闭。
- 右键菜单 `语音设置` 可调语速、音量、音色名并试听。
- 开启后会在AI回复时自动朗读文本。

## 10. 自动读取屏幕内容吐槽
- 右键菜单新增 `屏幕吐槽开关`，可随时开/关。
- 开启后会按 `settings.json -> screen_roast` 周期读取屏幕并自动吐槽（默认 25 秒轮询，90 秒冷却）。
- 读取优先级：
  1) OCR 读取屏幕文字（需要 `pytesseract` + 本机安装 Tesseract OCR）
  2) OCR 不可用时，退化为读取当前前台窗口标题
- 可在 `settings.json` 调整：
  - `enabled`：是否开启
  - `interval_seconds`：轮询间隔
  - `cooldown_seconds`：同类内容吐槽冷却
  - `ocr_lang`：OCR语言（如 `chi_sim+eng`）
  - `tesseract_cmd`：`tesseract.exe` 绝对路径（可空）

## 11. 系统音频转文字吐槽（回环录音）
- 右键菜单新增 `音频吐槽开关`，可随时开/关。
- 开启后会周期抓取"电脑正在播放的声音"（不是麦克风），先做语音转文字，再自动吐槽。
- 已支持实时模式：听到一段就先输出"实时听写"和"实时点评"，最后再输出一条"最终"汇总。
- 配置项在 `settings.json -> audio_roast`：
  - `enabled`：是否开启
  - `interval_seconds`：轮询间隔
  - `sample_seconds`：每次录音时长
  - `cooldown_seconds`：同类内容吐槽冷却
  - `min_chars` / `max_chars`：转写文本长度阈值
  - `realtime`：是否分段实时输出（默认 `true`）
  - `language`：识别语言（中文建议 `zh`）
  - `model_size`：Whisper模型大小（默认 `tiny`）

## 12. Nanobot 生信/爬数对话指令
- 需先在右键菜单打开 `Nanobot 引擎开关`。
- 对话前缀：
  - 生信任务：`/bio 你的需求` 或 `生信: 你的需求`
  - 固定数据库抓取：`/db 你的需求` 或 `/crawl 你的需求` 或 `爬数: 你的需求`
- 约定：面向医学生学习用途，避免临床诊疗建议；涉及医学或生物信息学事实时，回答末尾需附可核查来源链接。
- 数据源约束可在 `settings.json -> nanobot` 里设置：
  - `fixed_sources`：固定数据库/站点列表（每行一个域名或URL）
  - `source_policy`：`fixed_only`（仅限固定来源）/ `prefer_fixed`（优先固定来源）/ `off`（不限制）

## 13. 论文搜索整理辅助工具（对话直连）
- 在对话中使用 `/paper` 指令与论文工具联动：
  - `/paper list`：查看论文列表
  - `/paper id=论文ID 你的问题`：指定论文提问
  - `/paper 论文ID::你的问题`：简写形式
  - `/paper 你的问题`：仅在索引里只有一篇论文时可自动使用
- 论文索引位置：`论文搜索整理辅助工具/mad-professor-public-main/mad-professor-public-main/output/papers_index.json`
- 若要启用论文工具的LLM回答，请在 `论文搜索整理辅助工具/.../config.py` 中配置 `API_BASE_URL` 与 `API_KEY`。

# 14. iGEM 团队智能助手

OC桌宠已集成 iGEM 团队智能助手功能，专为 iGEM 竞赛团队设计，帮助不熟悉生信的成员也能使用基本工具，同时追踪组会进度、管理团队任务、快速访问技术文档。

## 14.1 功能总览

| 模块 | 功能 | 数据文件 |
|------|------|----------|
| 会议进展追踪器 | 记录组会、AI结构化摘要、检索、跨会议追踪 | `meetings.json` |
| 简化生信工作流 | 5种引导式生信工具+AI解读 | `bio_workflows_state.json` |
| 技术文档中心 | 分类索引、搜索、监视文件夹自动索引 | `doc_hub_index.json` |
| 团队任务看板 | 成员管理、任务分配、截止提醒 | `task_board.json` |

## 14.2 配置

在 `settings.json -> igem_assistant` 中配置：

```json
"igem_assistant": {
    "enabled": true,
    "team_name": "",
    "meeting_auto_push_feishu": true,
    "doc_watch_folders": [],
    "doc_auto_index": true,
    "doc_index_interval_minutes": 60,
    "task_deadline_remind_hours_before": 24,
    "task_deadline_push_feishu": true,
    "bio_api_url": "http://127.0.0.1:18901",
    "bio_api_timeout": 120
}
```

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `enabled` | 是否启用iGEM助手 | `true` |
| `team_name` | 团队名称 | `""` |
| `meeting_auto_push_feishu` | 会议记录自动推送飞书 | `true` |
| `doc_watch_folders` | 监视文件夹列表 | `[]` |
| `doc_auto_index` | 是否自动索引新文档 | `true` |
| `doc_index_interval_minutes` | 自动索引间隔（分钟） | `60` |
| `task_deadline_remind_hours_before` | 截止提前提醒时间（小时） | `24` |
| `task_deadline_push_feishu` | 截止提醒推送飞书 | `true` |
| `bio_api_url` | 生信API地址 | `http://127.0.0.1:18901` |
| `bio_api_timeout` | 生信API超时（秒） | `120` |

也可通过右键菜单 -> **iGEM助手** -> **iGEM助手设置** 图形化修改。

## 14.4 简化生信工作流

引导式多步工作流，让不熟悉命令行的iGEM成员也能使用基本生信工具。每个工作流分步引导：选择参数 -> 确认 -> 执行 -> AI解读结果。

### 预设工作流

| 工作流 | 触发词 | 步骤 | 需要的工具 |
|--------|--------|------|------------|
| 检查测序质量 (FastQC) | 测序质量 / fastqc / 质控 / qc | 选文件 → 选输出目录 → 确认 → 解读 | `fastqc` |
| 序列比对 (BLAST) | 比对 / blast / blastn / 同源 | 输入序列 → 选数据库 → 确认 → 解读 | `blastn` |
| 序列映射 (BWA MEM) | 映射 / mapping / bwa / 基因组比对 | 选参考基因组 → 选reads → 确认 → 解读 | `bwa` |
| 设计引物 (Primer3) | 引物 / primer / pcr引物 / primer3 | 输入序列 → 设参数 → 确认 → 解读 | `primer3_core` 或 `primer3-py` |
| 系统发育树 | 进化树 / phylogenetic / 建树 | 选比对文件 → 选方法 → 确认 → 解读 | `biopython` |

### 对话命令

| 命令 | 功能 |
|------|------|
| `/flow 帮我检查测序质量` | 启动FastQC工作流 |
| `/flow list` | 列出所有可用工作流 |
| `/flow cancel` | 取消当前工作流 |
| `帮我做blast比对` | 自然语言也能触发（无需/flow前缀） |

### 交互流程

以 FastQC 为例：

```
你：/flow 帮我检查测序质量
桌宠：已启动工作流：检查测序质量 (FastQC)
     步骤 1/4：请选择你的测序文件路径（fastq/fasta格式）
     （输入 /flow cancel 可随时取消）

你：C:/data/reads_R1.fastq.gz
桌宠：[检查测序质量 (FastQC)] 步骤 2/4：选择输出目录（留空则与输入文件同目录）

你：C:/data/fastqc_output
桌宠：[检查测序质量 (FastQC)] 步骤 3/4：确认运行 FastQC 质量检查？

你：确认
桌宠：执行完成！
     结果：...
     --- AI解读 ---
     测序质量整体良好，Per base sequence quality大部分在绿色区域...
```

### 工具安装

生信工作流需要对应工具已安装并在系统PATH中：

| 工具 | 安装方式 |
|------|----------|
| FastQC | `conda install -c bioconda fastqc` 或从 https://www.bioinformatics.babraham.ac.uk/projects/fastqc/ 下载 |
| BLAST+ | `conda install -c bioconda blast` 或从 NCBI FTP 下载 |
| BWA | `conda install -c bioconda bwa` 或从 http://bio-bwa.sourceforge.net/ 下载 |
| Primer3 | `pip install primer3-py` 或从 https://primer3.org 下载 |
| 系统发育树 | `pip install biopython`（内置NJ建树） |

### 图形界面

右键菜单 -> **iGEM助手** -> **生信工作流**：打开向导窗口，点击工作流名称启动。

## 14.5 技术文档中心

分类索引团队的技术文档，支持搜索、自动索引、LLM摘要。

### 文档分类

| 分类 | 图标 | 说明 |
|------|------|------|
| 湿实验Protocol | 🧪 | 质粒提取、PCR、转化等实验操作流程 |
| 干实验工具 | 💻 | 生信分析脚本、工具使用指南 |
| 安全规范 | ⚠️ | 实验室安全规定、废弃物处理 |
| 比赛规则 | 📋 | iGEM竞赛规则、评审标准 |
| Wiki | 🌐 | 团队Wiki页面内容 |
| 会议记录 | 📝 | 组会记录文档 |
| 教程 | 📖 | 技术教程、学习资料 |
| 其他 | 📁 | 未归类的文档 |

### 支持的文件格式

`.txt` `.md` `.pdf` `.docx` `.py` `.fasta` `.fa` `.fastq` `.gb` `.gff` `.sam` `.bam` `.vcf`

### 对话命令

| 命令 | 功能 |
|------|------|
| `/doc 质粒提取` | 搜索相关文档 |
| `/doc 文档中心` | 显示各分类文档数量 |
| `/doc 列表` | 列出所有分类 |
| `/doc 添加 C:/docs/protocol.md 湿实验Protocol` | 添加文档到指定分类 |
| `/doc 监视 C:/iGEM/docs` | 设置监视文件夹 |
| `/doc 扫描` | 手动触发扫描 |

### 监视文件夹自动索引

1. 设置监视路径：`/doc 监视 C:/iGEM/docs`
2. 程序每小时自动扫描该文件夹
3. 新增或修改的文件会被自动索引
4. 分类根据文件夹名称自动推断（如 `wetlab/` 下自动归类为湿实验）

自动推断规则：

| 文件夹名包含 | 自动分类 |
|-------------|----------|
| wetlab / protocol | 湿实验Protocol |
| drylab / tool / scripts | 干实验工具 |
| safety / safe | 安全规范 |
| rules / igem_rules | 比赛规则 |
| wiki | Wiki |
| meeting / meetings | 会议记录 |
| tutorial / guide / learn | 教程 |

生信格式文件（`.fasta` `.fastq` `.gb` `.sam` `.bam` `.vcf` `.py`）也会自动归类为干实验工具。

### 图形界面

右键菜单 -> **iGEM助手** -> **文档中心**：分类浏览、搜索、添加文档、设置监视文件夹、手动扫描。

| 进行中 | 🔨 | 正在执行的任务 |
| 已完成 | ✅ | 已完成的任务 |
| 卡住 | 🚧 | 遇到障碍的任务 |
| 已取消 | ❌ | 不再需要的任务 |

### 优先级

| 级别 | 中文 | 说明 |
|------|------|------|
| low | 低 | 不紧急 |
| medium | 中 | 默认优先级 |
| high | 高 | 需要优先处理 |
| urgent | 紧急 | 必须立即处理 |

### 截止提醒

- 每小时整点自动检查任务截止日期
- 在 `task_deadline_remind_hours_before` 设定的小时数内到期时，自动弹出提醒
- 已过期任务也会被提示
- 开启 `task_deadline_push_feishu` 时同步推送飞书

### 会议联动

保存会议记录后，摘要中的"下一步"会自动建议创建任务：

```
保存会议后显示：
建议创建的任务（2条）：
  · 尝试pET28a载体
  · 周五前完成凝胶电泳
```

### 图形界面

右键菜单 -> **iGEM助手** -> **任务看板**：看板视图（待办/进行中/已完成/卡住四列），支持一键完成任务。

右键菜单 -> **iGEM助手** -> **团队成员**：成员列表，显示每位成员的角色、技能和当前任务。

## 14.7 飞书联动

iGEM助手与飞书集成，支持以下事件推送：

| 事件 | 触发条件 | 推送内容 |
|------|----------|----------|
| `meeting_summary` | 保存会议记录 | 结构化摘要 |
| `task_deadline` | 任务即将到期 | 截止提醒 |
| `reminder` | 日程提醒到期 | 提醒文本 |

前提：需先在右键菜单打开 **飞书桥接开关**，并在 `settings.json -> feishu` 中配置 `appId` 和 `appSecret`。

## 14.8 数据文件说明

| 文件 | 对应模块 | 格式 |
|------|----------|------|
| `meetings.json` | 会议进展追踪器 | 会议记录数组 |

所有数据文件均为 JSON 格式，可直接编辑。程序运行时自动读写，无需手动操作。
